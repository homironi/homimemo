## エラー

```txt
Caused by: Error: D1_ERROR: too many SQL variables at offset 434: SQLITE_ERROR

Caused by: Error: too many SQL variables at offset 434: SQLITE_ERROR
```

D1のパラメータ数制限に引っかかるエラーです。  
`434`がパラメータ数ですね。

<CardPreviewUrl url="https://developers.cloudflare.com/d1/platform/limits/" />
<CardPreviewUrl url="https://github.com/drizzle-team/drizzle-orm/issues/2479" />

## 先にコード全体

```ts
import { SQLiteTable, SQLiteInsertValue, SQLiteInsertBase } from "drizzle-orm/sqlite-core";

import { DrizzleD1Database } from "drizzle-orm/d1";

import * as schema from "./schema";

export type Database = DrizzleD1Database<typeof schema> & {
  $client: D1Database;
};

type InsertQuery<T extends SQLiteTable> = SQLiteInsertBase<
  T,
  "async",
  D1Result<unknown>
>;

const D1_SQL_MAX_PARAMETERS = 100;

/**
 * D1のパラメータ数制限（100）を考慮したchunk分割インサート
 * カラム数から自動的にチャンクサイズを計算する
 *
 * https://developers.cloudflare.com/d1/platform/limits/
 *
 * https://github.com/drizzle-team/drizzle-orm/issues/2479
 */
export async function chunkedInsert<T extends SQLiteTable>(
  db: Database,
  table: T,
  values: SQLiteInsertValue<T>[],
  onInserted?: (insert: InsertQuery<T>) => InsertQuery<T>,
) {
  if (values.length === 0) return;

  const columnCount = Object.keys(table).filter(
    key => !key.startsWith("_") && !key.startsWith("$"),
  ).length;
  const chunkSize = Math.max(1, Math.floor(D1_SQL_MAX_PARAMETERS / columnCount));

  const chunks = Array.from(
    { length: Math.ceil(values.length / chunkSize) },
    (_, i) => values.slice(i * chunkSize, (i + 1) * chunkSize),
  );

  const batches = chunks.map((chunk) => {
    const query = db.insert(table).values(chunk);
    const q = onInserted ? onInserted(query) : query;
    return q;
  });
  const [firstBatch, ...restBatches] = batches;
  if (!firstBatch) throw new Error("chunkedInsertQueries length must be 1 or greater");

  await db.batch([firstBatch, ...restBatches]);
}
```

## ちょいメモ

基本的にはそのままの通りです。  
カラム数を計算して、D1の100の制限に合うように chunk サイズを計算して、chunk に分割して、D1で使える`db.batch`にぶん投げてます。  

<TextBlock blockType="warning">
あくまでテーブルの定義からカラム数を計算しているので、部分Insertなどの場合にはもう少し chunk を増やせる可能性があります。そこまでは対応していません。
</TextBlock>

特筆すべきは以下の内容でしょうか。

```ts
const [firstBatch, ...restBatches] = batches;
if (!firstBatch) throw new Error("chunkedInsertQueries length must be 1 or greater");
```
ここは `db.batch` のための型合わせです。`[BatchItem<"sqlite">, ...BatchItem<"sqlite">[]]`に合わせるためにえいやえいやして合わせてます。  

```ts title="batches を直入れすると型が合わない"
const batches = chunks.map((chunk) => {
  const query = db.insert(table).values(chunk);
  const q = onInserted ? onInserted(query) : query;
  return q;
});

await db.batch([...batches]);
```

以下のエラーが出ます。

```txt
型 'InsertQuery<T>[]' の引数を型 'readonly [BatchItem<"sqlite">, ...BatchItem<"sqlite">[]]' のパラメーターに割り当てることはできません。
ソースには、ターゲットの位置 0 にある必須要素と一致するものがありません。
```

`const [firstBatch, ...restBatches] = batches;`で分割することで1つ目の要素があることを保証した型になって合うようです。こういう書き方（型）あるんですね。  
TypeScriptは型が柔軟でおもしろいです。

## テスト

Vitestでテストも書きました。  
`chunkedInsert`のテストなのにテストデータ生成で`chunkedInsert`を使っているのは御愛嬌ということで……。

こちらは import があちこちからきてて申し訳ないですが、相対パス系は全てプロジェクト固有のものです。  
それぞれのプロジェクトのものに置き換えて考えていただけると幸いです。

```ts
import { env } from "cloudflare:workers";
import { getTableName } from "drizzle-orm";
import { SQLiteInsertValue } from "drizzle-orm/sqlite-core";
import { describe, expect, it, beforeAll } from "vitest";

import { createDb } from "../../db";
import {
  feedTable,
  folderTable,
  postTable,
  readLaterTable,
  subscriptionTable,
} from "../../db/schema";
import { chunkedInsert } from "../../db/utils";
import { defaultTestSession } from "../setup";

// 外部キー依存のある前提データ
const SEED = {
  userId: defaultTestSession.user.id,
  feedId: "bulk-insert-test-feed",
  folderId: "bulk-insert-test-folder",
  postId: "bulk-insert-test-post-0",
} as const;

const testCases = [
  {
    label: getTableName(feedTable),
    table: feedTable,
    generateTestData: (length: number) =>
      Array.from({ length }, (_, i) => ({
        id: `bulk-feed-${i}`,
        url: `https://example.com/feed/${i}`,
        siteTitle: `feed title ${i}`,
        siteUrl: "https://example.com/",
        updatedAt: new Date(),
      })) satisfies SQLiteInsertValue<typeof feedTable>[],
  },
  {
    label: getTableName(folderTable),
    table: folderTable,
    generateTestData: (length: number) =>
      Array.from({ length }, (_, i) => ({
        id: `bulk-folder-${i}`,
        userId: SEED.userId,
        parentId: null,
        name: `folder ${i}`,
      })) satisfies SQLiteInsertValue<typeof folderTable>[],
  },
  {
    label: getTableName(postTable),
    table: postTable,
    generateTestData: (length: number) =>
      Array.from({ length }, (_, i) => ({
        id: `bulk-post-${i}`,
        feedId: SEED.feedId,
        url: `https://example.com/post/${i}`,
        title: `post title ${i}`,
        description: `post description ${i}`,
        publishedAt: new Date(Date.now() - i * 1000),
      })) satisfies SQLiteInsertValue<typeof postTable>[],
  },
  {
    label: getTableName(subscriptionTable),
    table: subscriptionTable,
    generateTestData: (length: number) =>
      Array.from({ length }, (_, i) => ({
        // subscriptionTableはfeedId+userIdが複合PKなので
        // feedIdをindex分けして複数行作れるようにbulk-sub-feed-${i}を用意
        feedId: `bulk-sub-feed-${i}`,
        userId: SEED.userId,
        folderId: null,
        title: `subscription ${i}`,
      })) satisfies SQLiteInsertValue<typeof subscriptionTable>[],
  },
  {
    label: getTableName(readLaterTable),
    table: readLaterTable,
    generateTestData: (length: number) =>
      Array.from({ length }, (_, i) => ({
        // readLaterTableはpostId+userIdが複合PKなので
        // postIdをindex分けして複数行作れるようにbulk-rl-post-${i}を用意
        postId: `bulk-rl-post-${i}`,
        userId: SEED.userId,
        savedAt: new Date(Date.now() - i * 1000),
      })) satisfies SQLiteInsertValue<typeof readLaterTable>[],
  },
] as const;

const lengths = [1, 10, 15, 16, 17, 20, 30, 50, 100];

describe("chunkedInsert", () => {
  // 外部キー制約を満たすための前提データを挿入
  beforeAll(async () => {
    const db = createDb(env.DB);

    // feedTable（postTable・subscriptionTableの前提）
    await chunkedInsert(db, feedTable, [
      {
        id: SEED.feedId,
        url: "https://example.com/feed/seed",
        siteTitle: "seed feed",
        siteUrl: "https://example.com/",
        updatedAt: new Date(),
      },
      // subscriptionTableのテスト用フィード（length最大分）
      ...Array.from({ length: Math.max(...lengths) }, (_, i) => ({
        id: `bulk-sub-feed-${i}`,
        url: `https://example.com/sub-feed/${i}`,
        siteTitle: `sub feed ${i}`,
        siteUrl: "https://example.com/",
        updatedAt: new Date(),
      })),
    ], q => q.onConflictDoNothing());

    // postTable（readLaterTableの前提）
    await chunkedInsert(db, postTable, [
      ...Array.from({ length: Math.max(...lengths) }, (_, i) => ({
        id: `bulk-rl-post-${i}`,
        feedId: SEED.feedId,
        url: `https://example.com/rl-post/${i}`,
        title: `rl post ${i}`,
        description: null,
        publishedAt: new Date(Date.now() - i * 1000),
      })),
    ], q => q.onConflictDoNothing());
  });

  describe.for(testCases)("$label", ({ label, table, generateTestData }) => {
    it.for(lengths)(`${label}: %i件insertできる`, async (length) => {
      const db = createDb(env.DB);
      const testData = generateTestData(length);

      await expect(
        chunkedInsert(db, table, testData, q => q.onConflictDoNothing()),
      ).resolves.not.toThrow();
    });
  });
});
```
