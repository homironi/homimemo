ソフトごとに合成モードの表記が違うことがあるので、それのまとめついでに計算内容などもまとめてみました。  
明るくなる、暗くなる、というざっくりしたことはわかるけど実際どんな計算で色が計算されているんだろう、ということを中心に書いています。

CLIP STUDIO PAINT をメインで使っているので、その表示を中心に書きます。

## 表示名

Affinityは他2つのソフトとは名前が結構違いました。  

Photoshopの「覆い焼き（リニア） - 加算」は「覆い焼き（リニア）」と表記されることも多いので、加算ということを忘れがち。

|CLIP STUDIO PAINT|Photoshop|Affinity|
|--|--|--|
|加算|覆い焼き（リニア） - 加算|追加|
|比較（暗）|比較（暗）|暗くする|
|比較（明）|比較（明）|明るくする|
|カラー比較（暗）|カラー比較（暗）|色を暗くする|
|カラー比較（明）|カラー比較（明）|色を明るくする|

## いろいろ前提など

基本的にCLIP STUDIO PAINT PRO での内容中心に書いています。  
ソフトによってない合成モードが出てきたり、操作の説明をする場合には違う操作が必要な場合があるので、ご注意ください。

### 全て共通の話

- 設定レイヤー：対象の合成モードを設定しているレイヤー
- RGBは0～255の範囲に収められます。
- 合成の結果、0を下回れば0に、255を超えたら255になります。

### 正規化

RGBがとる値の範囲は0～255です。  
しかし、乗算や除算などではそのままRGBの値を計算すると値の大きさが変わってしまいます。  
この大きさが変わらないように調整することを正規化と呼びます。

<Accordion summary="乗算で説明してみます（※長いので折りたたみ）">

細かいところは乗算の項目で書いているので、今は「乗算は色の値を掛け算して255で割った値がRGBの各値になる」と理解していてください。

RGBを分数として考えてみるとわかりやすいかもしれません。  
`rgb(250, 250, 250)`と`rgb(100, 100, 100)`を例にします。

R100というのは0～255のうち、100の量Rがあるということです。  
この例えでいくとR100は255分の100という値だと考えることができます。  
R250なら255分の250です。

これを分数で掛け算したとき、255分の250 * 255分の100 = （255の2乗）分の25000（25000 / 255 * 255）です。  
この計算をより簡単に行うには、255を掛けておくことによって 250 * 100 とすることができます。計算しやすくなりました。  
そして、250 * 100で得た25000という値は255を先に掛けておいたのでもとの大きさにもどすために255で割ります。  
計算をRGBの値0～255でそのままやる代わりに、最後に255で割りました。  
</Accordion>

すなわち「**計算しやすい大きさで計算したあと本来の大きさに戻す**」ということを正規化とこの記事では呼びます。   
**センチメートルで計算して、メートルに直すみたいな**感覚です。

### 色調補正

この記事ではレイヤーの合成モードの話ですが、関係ありそうな色調補正レイヤーを紹介します。  
内容はCLIP STUDIO PAINTの内容ですが、他のソフトにも同様の調整ができる方法があると思うので、必要な場合は調べてみてください。

#### 色の反転

色を反転させることができます。  
合成モードの話の中でも「色を反転させてそれをうんたらかんたら～」という説明も出てきます。  
実際にどうなるんだろう、と手元で確認したい場合は「レイヤー右クリック ＞ 新規色調補正レイヤー ＞ 階調の反転」で確認できます（CLIP STUDIO PAINT）。  

色調の反転レイヤー以下全てに反映されますが、他のレイヤーと同じようにクリッピングやマスクなどで範囲を限定することもできます。

## 通常

レイヤーの順番にそのまま重なる通常のレイヤー。

## 加算

下のレイヤーの色をこの設定がされたレイヤーの色でチャンネルごとに加算（足し算）します。  

白（`rgb(255, 255, 255)`）を足すと255を超えられないので、下のレイヤー色に関わらず白になります。  

黒（`rgb(0, 0, 0)`）を足すと0を足すことになるので、下のレイヤー色がそのまま表示されます。

設定レイヤー：`rgb(20, 100, 20)`

## 減算

下のレイヤーの色からこの設定がされたレイヤーの色をチャンネルごとに減算（引き算）します。  

白（`rgb(255, 255, 255)`）を引くと0を下回れないので、下のレイヤー色に関わらず黒になります。  

黒（`rgb(0, 0, 0)`）を引くと引かれる値が0なので、下のレイヤー色がそのまま表示されます。

## 乗算

この設定がされたレイヤーと下のレイヤーの色をチャンネルごとに乗算します。  
掛け算した値を255で割った値が結果のRGB値です。

もとの色より暗い色になります。  

白（`rgb(255, 255, 255)`）を合成すると（掛け算後に255で割って正規化することを考慮すると）1を掛けるのと同じなので、下のレイヤーの色がそのまま表示されます。  

黒（`rgb(0, 0, 0)`）を合成すると（掛け算後に255で割って正規化することを考慮すると）0を掛けるのと同じなので、下のレイヤー色に関わらず黒になります。  
陰影などによく使います。

設定レイヤー：`rgb(20, 100, 20)`

下のレイヤー：`rgb(10, 20, 220)`  
<CardPreviewUrl url="https://www.colordic.org/colorsample/0a14dc" />

## 除算

この設定がされたレイヤーと下のレイヤーの色をチャンネルごとに除算します。  
割り算した値に255をかけます。

乗算と逆で、除算をするので明るい色を得やすいです。

白（`rgb(255, 255, 255)`）を合成すると（割り算後に255を掛けて正規化することを考慮すると）1で割るのと同じなので、下のレイヤーの色がそのまま表示されます。  

黒（`rgb(0, 0, 0)`）を合成すると（割り算後に255を掛けて正規化することを考慮すると）0で割るのと同じ。いわゆる0除算（矛盾なく定義できない計算）なのでソフトによっては怪しい挙動が確認できる。だいたい下のレイヤー色に関わらず白になります。  
`rgb(1, 1, 1)`だと割り算後の値は変わらず、その後正規化のために255を掛けるので、下のレイヤー色に関わらず白になります。

## コントラスト系

色が明るいか暗いかで、設定レイヤーモードごとに応じた計算がされます。  

CLIP STUDIO PAINTでは明記されているのをみつけられませんでしたが、PhotoshopやAffinityでは「50%のグレー」より明るいか暗いかという判別をしているようです。  

Affinityでは[「グレーが50％ちょうどの色は変化しません」](https://www.affinity.studio/ja_jp/help/layers-layer-blend-modes/#heading------------)と記されていました。  
実際CLIP STUDIO PAINTなどでも50%のグレーだとコントラスト系の合成モードでは下のレイヤーの色がそのまま表示されます。

### オーバーレイ

下のレイヤーの色を基準に、暗い色を重ねるとより暗く、明るい色を重ねるとより明るくなります。

下のレイヤーの黒（`rgb(0, 0, 0)`）と白（`rgb(255, 255, 255)`）はそれぞれより暗くなることもより明るくなることもできないので、そのまま表示されます。

CLIP STUDIO PAINT では暗くする際は乗算、明るくする際はスクリーンのように合成されます。

### ソフトライト

この設定がされたレイヤーの輝度によって結果が変わります。

CLIP STUDIO PAINT では暗くする際は焼き込み、明るくする際は覆い焼きのように合成されます。

### ハードライト

この設定がされたレイヤーの輝度によって結果が変わります。

CLIP STUDIO PAINT では暗くする際は乗算、明るくする際はスクリーンのように明るくなります。

### ビビッドライト

この設定がされたレイヤーの色によって結果が変わります。

CLIP STUDIO PAINT では暗くする際は焼き込みカラー、明るくする際は覆い焼きカラーの効果が適用されます。

### リニアライト

この設定がされたレイヤーの色によって結果が変わります。

CLIP STUDIO PAINT では暗くする際は焼き込み（リニア）、明るくする際は加算の効果が適用されます。

### ピンライト

この設定がされたレイヤーの色によって結果が変わります。

CLIP STUDIO PAINT では以下のようになります。  
設定レイヤーの色が暗く下のレイヤーがそれより明るい部分は比較（暗）、設定レイヤーよりも明るい色は変更されない。  
設定レイヤーの色が明るく下のレイヤーがそれより暗い色の部分なら比較（明）、設定レイヤーよりも暗い色は変更されない。

### ハードミックス

コントラスト系として分類しましたが、これだけは他と違う計算がされます。50%グレーはここでは忘れてください。

レイヤーを加算合成した後、255以上なら255、255未満なら0に値が置き換えられます。  
つまりRGB値が0か255のどちらかの値になり、以下の8色のいずれかになります。

- `rgb(0, 0, 0)`
- `rgb(255, 0, 0)`
- `rgb(255, 255, 0)`
- `rgb(0, 255, 0)`
- `rgb(0, 255, 255)`
- `rgb(0, 0, 255)`
- `rgb(255, 0, 255)`
- `rgb(255, 255, 255)`
