Cloudflare Workers、Cloudflare D1のテストをVitestで書いた際に、`vi.mock()`が効かずモックではなく実際の処理が呼ばれてしまうことがあったのでメモ。

## 環境

- `@cloudflare/vitest-pool-workers` 0.16.10
- `vitest` 4.1.7
- Cloudflare D1

## だめだったテストのざっくりイメージ

<CardPreviewUrl url="https://github.com/cloudflare/workers-sdk/issues/10201" />

GitHubのイシュー同様、`setup.ts`でD1のマイグレーションをテストで適用するようにしていました。

```ts title=setup.ts
import { applyD1Migrations } from "cloudflare:test";
import { env } from "cloudflare:workers";

await applyD1Migrations(env.DB, env.TEST_MIGRATIONS);
```

内容自体は単純で、Cloudflareの公式で説明されている通りに書いています。

<CardPreviewUrl url="https://developers.cloudflare.com/workers/testing/vitest-integration/test-apis/#d1" />
<CardPreviewUrl url="https://github.com/cloudflare/workers-sdk/tree/main/fixtures/vitest-pool-workers-examples/d1" />

## 回避策

- GitHubのイシュー同様：`vi.mock()`を書く前に`vi.resetModules();`
- mockするものを使うものはその後にDynamic import する：  
`const { default: routes } = await import("../worker/routes");`

`routes`の中で使う`isHoge`をmockしたい場合は、`vi.resetModules()`をしたうえで`routes`をDynamic importすることでmockが効くようになりました。

```ts
// import ...
import { isHoge } from "../worker/lib/hoge";

// mockが効かなくなる既知バグ回避：https://github.com/cloudflare/workers-sdk/issues/10201
vi.resetModules();
const { default: routes } = await import("../worker/routes");

vi.mock(import("../worker/lib/hoge"), () => ({
  isHoge: vi.fn(),
}));

const mockIsHoge = vi.mocked(isHoge);
```
